Nextra has been installed as release "{{ .Release.Name }}".

To check the status of the rollout:
  kubectl get pods -n {{ .Release.Namespace }} \
    -l app.kubernetes.io/instance={{ .Release.Name }}

To reach the app:
{{- if .Values.ingress.enabled }}
  Add this to /etc/hosts (or your DNS):
    <INGRESS_IP>  {{ .Values.ingress.host }}
  Then open:
    http{{- if .Values.ingress.tls.enabled }}s{{- end }}://{{ .Values.ingress.host }}/app/en
{{- else }}
  Ingress is disabled. Port-forward the portal service:
    kubectl port-forward svc/{{ include "nextra.fullname" . }}-portal 8080:{{ .Values.portal.port }}
  Then open http://localhost:8080/app/en
{{- end }}

Core services deployed:
  - backend        (Drogon C++ API)
  - frontend       (Next.js 16)
  - job-scheduler  (Kafka-backed async jobs)
  - cron-manager   (scheduled tasks)
  - sso            (single sign-on)
  - portal         (nginx reverse proxy)
  - postgres, redis, kafka, elasticsearch

Optional add-on services can be enabled incrementally in
values.yaml under `optionalServices.*.enabled`.
